## Skill hygiene
When you notice you've carried out the same multi-step workflow more than
twice in this project (a recurring ticket format, a build/deploy sequence,
a comms template), pause and propose codifying it as a skill:
- Draft a SKILL.md (name + a tightly-scoped "Use when…" description).
- Show me the draft and where it'd live (.claude/skills/<name>/).
- Only write the file after I confirm. No speculative or one-off skills.
